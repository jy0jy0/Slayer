# 테스트 데이터

이력서 파싱, JD 매칭, 이력서 최적화 기능 테스트를 위한 샘플 데이터입니다.

## 이력서 샘플 (5건)

| 파일 | 포맷 | 페르소나 | 경력 | 전문분야 | 프로젝트 수 |
|------|------|----------|------|----------|------------|
| `backend_김준혁.md` | Markdown | 김준혁 (30세) | 5년 | Backend / Java Spring | 4개 |
| `ml_이서연.json` | JSON (nested) | 이서연 (27세) | 3년 | ML/AI / Python PyTorch | 3개 |
| `frontend_박민재.md` | Markdown | 박민재 (33세) | 7년 | Frontend / React TS | 5개 |
| `devops_최은지.txt` | Plain text | 최은지 (29세) | 4년 | DevOps / AWS K8s | 4개 |
| `fullstack_정하늘.json` | JSON (flat) | 정하늘 (26세) | 2년 | Fullstack / Node React | 3개 |

## 포맷 다양성

파싱 모듈의 다양한 입력에 대한 대응력을 테스트하기 위해 의도적으로 다르게 작성:

- **Markdown 2건**: 섹션 순서 상이 (전통적 순서 vs 스킬-퍼스트)
- **JSON 2건**: 구조 상이 (영문 키 nested vs 한글 키 flat)
- **Plain text 1건**: 최소 포맷팅, `---` 구분자만 사용
- **분량**: 짧은(정하늘) ~ 매우 긴(박민재)

## 사용처

- **예신**: 이력서 파싱 Pipeline 테스트 입력
- **현지**: JD-이력서 매칭 테스트
- **지호**: 이력서 최적화 Agent 테스트

## 주의사항

- 모든 데이터는 가상의 인물입니다
- 회사명은 실제 회사를 사용했으나, 경력 내용은 허구입니다
